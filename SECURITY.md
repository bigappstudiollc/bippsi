# Security Policy

## Scope

This policy covers the open specifications and reference implementations in this repository. The bippsi.com application code is private and out of scope.

## Reporting Vulnerabilities

If you find a security issue in a Bippsi specification or reference implementation, please report it responsibly:

- **Contact:** https://bippsi.com/contact
- **Response time:** We acknowledge reports within 3 business days
- **Disclosure:** We coordinate disclosure timing with the reporter. We ask for 90 days before public disclosure to allow us to address the issue.

## What to Report

- Flaws in protocol specifications that could lead to token theft, replay attacks, or unauthorized access
- Vulnerabilities in reference implementations (even though they are examples, we want them to be safe to copy)
- Issues where the spec could be implemented in a way that creates security problems

## What Not to Report

- Bugs in the bippsi.com application (use https://bippsi.com/contact directly)
- General questions about how the protocol works (open a GitHub issue instead)
- Social engineering or phishing concerns unrelated to the specifications
